## 1. 엑셀 컬럼 구조 → 프로그램 입력 항목 변환

엑셀 시트별로 예상 컬럼을 프로그램의 **입력 필드(속성)** 로 매핑한다.  
실제 엑셀 시트명·컬럼명이 다르면 이 표를 기준으로 매핑만 조정하면 된다.

| 방문처 / 고객 | `customer_id` (또는 고객명) | FK / string | ⬜ | 고객 마스터 연결 |

| 작업 내용 | `content` | text | ✅ | 업무 내용 요약 |
| 담당자(작성자/방문자) | `assignee_id` (또는 담당자명) | FK / string | ⬜ | 담당자 마스터 연결 |

## 2. 공통 필드 체계화 (날짜, 담당자, 업무내용, 진행상태, 비고, 첨부)

| 분류 | 필드(영문) | 용도 | 타입 | 일지 | 점검 | AS | 고객 | 장비 | 담당자 |
|------|------------|------|------|------|------|-----|------|------|--------|
| **날짜** | `date` / `*_date` | 기준일·접수일·완료일 등 | date | ✅ | ✅(예정일) | ✅ | — | ⬜(설치일) | — |

| **업무내용** | `content` | 작업/요청 내용 | text | ✅ | — | ✅ | — | — | — |
| **진행상태** | `status` | 진행중/완료/보류 등 | enum | ⬜ | — | ✅ | — | — | — |
| **비고** | `remarks` | 자유 메모 | text | ⬜ | ⬜ | ⬜ | ⬜ | ⬜ | ⬜ |
| **첨부** | `attachments` | 파일 경로·ID 목록 | list/JSON | ⬜ | — | ⬜ | — | — | — |

- **진행상태**  
  - 일지: 선택 사항. 예) `진행중` / `완료` / `보류` / `미입력`.

  - 엑셀 내보내기 시: "첨부 있음" 플래그 또는 파일명 목록을 한 셀에 텍스트로 넣는 방식 권장.

## 3. 엑셀 저장 형식 유지 vs DB/CSV 병행 저장 방안

### 3-1. 권장: DB(또는 JSON) 주 저장 + 엑셀은 내보내기 전용

- **주 저장소:** SQLite 또는 JSON/CSV 세트  
  - 검색·필터·월별 집계·통계에 유리.  
  - openpyxl/pandas로 **읽기/쓰기**는 가능하지만, 엑셀을 “실시간 저장소”로 쓰면 동시 편집·잠금·대용량 시 성능 이슈가 있음.
- **엑셀:** “내보내기”로만 사용  
  - 주기적 백업, 부서 공유, 기존 양식 호환용.  
  - 시트별로 **일지 / 점검 / AS / 고객 / 장비 / 담당자**를 각각 시트로 내보내면 실무 사용성 유지.

### 3-2. 옵션 A: SQLite 주 저장 + 시트별 엑셀 내보내기

| 저장소 | 역할 | 도구 |
|--------|------|------|
| SQLite | 일상 입력·수정·조회·검색·통계 | sqlite3, SQLAlchemy 등 |
| 엑셀 파일 | 백업·공유·기존 양식 제출 | openpyxl로 시트별 생성 |

- **장점:** 구조화된 조회·통계·확장에 가장 유리.  
- **내보내기:** pandas로 테이블 읽어서 `to_excel(sheet_name=...)` 또는 openpyxl로 워크북에 시트 추가.

### 3-3. 옵션 B: JSON/CSV 세트 주 저장 + 엑셀 내보내기

| 파일 | 역할 |
|------|------|
| `journal.json` 또는 `journal_YYYYMM.csv` | 일지 |
| `inspections.json` | 점검 일정 |
| `as_requests.json` | AS |
| `customers.json` | 고객 |
| `equipment.json` | 장비 |
| `assignees.json` | 담당자 |

- **장점:** 버전 관리·백업이 단순하고, pandas로 읽어서 엑셀/통계 처리 용이.  
- **단점:** 관계(예: customer_id) 검증·참조 무결성은 코드로 직접 관리해야 함.

### 3-4. openpyxl / pandas 활용 포인트

- **openpyxl:**  
  - 엑셀 **내보내기** 시 서식(헤더 굵게, 열 너비, 날짜 형식) 유지할 때 사용.  
  - 기존 엑셀 양식에 데이터만 채우는 “서식 맞춤 내보내기”(2차)에도 적합.
- **pandas:**  
  - DB/CSV/JSON → DataFrame → `to_excel()` 로 시트 생성.  
  - 월별·기간별 필터 후 엑셀 한 시트로 내보내기, 통계 요약 시트 추가 등에 활용.

**정리:**  
- **1차:** SQLite(또는 JSON) 주 저장 + 시트별 엑셀 내보내기(pandas/openpyxl).  
- **엑셀 파일을 실시간 저장소로 쓰는 방식은 비권장** — 호환용·백업용으로만 사용.

## 4. 검색·필터·월별 정리·출력 구조 설계

### 4-1. 검색

| 대상 | 검색 필드 | 구현 관점 |
|------|-----------|-----------|
| 일지 | 날짜, 고객명, 작업구분, 내용(텍스트), 담당자 | `date` 범위, `customer_id`/이름, `work_type`, `content` LIKE, `assignee_id` |
| AS | 접수일, 고객, 상태, 내용 | `received_date`, `customer_id`, `status`, `content` LIKE |
| 점검 | 고객, 예정일, 담당자 | `customer_id`, `next_due_date`, `assignee_id` |
| 고객/장비/담당자 | 이름·연락처 등 | `name`, `phone` 등 LIKE |

- DB 사용 시: 위 필드에 인덱스(date, customer_id, status 등) 두면 검색 속도 유리.  
- **전체 통합 검색(2차):** 일지+AS 내용을 한 번에 검색하려면 검색용 뷰(VIEW) 또는 통합 검색 API에서 여러 테이블 UNION 후 필터.

### 4-2. 필터

- **공통:**  
  - 기간(시작일–종료일).  
  - 고객(드롭다운 또는 검색).  
- **일지:** 작업 구분(점검/AS/영업/기타), 진행상태(선택 시), 담당자.  
- **AS:** 상태(접수/진행중/완료/보류), 우선순위.  
- **점검:** 예정일 범위, 담당자.

필터 조건은 **쿼리 파라미터 또는 DTO**로 넘기고, SQL/DataFrame 필터로 적용.  
엑셀 내보내기 시에도 “현재 목록(필터 결과만)” 내보내기 옵션을 두면 실무 활용도가 높다.

### 4-3. 월별 정리

- **저장:** 데이터는 **일자/일시 단위**로만 저장하고, “월”은 조회 시 계산.  
  - 예: `strftime('%Y-%m', date)` 또는 pandas `df['month'] = pd.to_datetime(df['date']).dt.to_period('M')`.
- **기능 제안:**  
  - “이번 달 일지” / “특정 월 일지” 조회.  
  - 월별 건수·고객별 건수 등 간단 통계(2차에서 통계 화면 확장).  
- **내보내기:** “2025년 2월 일지”만 필터해서 한 시트로 엑셀 출력.

### 4-4. 출력용 구조

| 출력 형태 | 데이터 단위 | 비고 |
|-----------|-------------|------|
| 화면 목록 | 페이지당 N건, 정렬(날짜 역순 등) | 목록 API/쿼리에 `ORDER BY`, `LIMIT/OFFSET` |
| 엑셀 시트 1장 | 일지/점검/AS 등 시트별 전체 또는 필터 결과 | pandas/openpyxl로 시트 생성 |
| 월별 보고서(1장) | 해당 월 일지 + (선택) 요약 행 | 상단에 요약(건수, 고객 수 등), 이하 상세 행 |
| 인쇄용 | 동일 데이터를 PDF/프린트 레이아웃으로 | 2차에서 레이아웃·PDF 출력 검토 |

- **공통:**  
  - “출력용”은 **조회 결과(필터·정렬 적용된 데이터)**를 그대로 사용.  
  - 날짜 형식(YYYY-MM-DD 또는 YYYY.MM.DD), 한글 인코딩(UTF-8), 엑셀 열 순서는 “엑셀 컬럼(예)”와 맞추면 실무 사용성 유지에 좋다.

## 6. 엑셀 시트 ↔ 프로그램 매핑 요약

| 엑셀 시트(예) | 프로그램 엔티티 | 내보내기 시 시트명 제안 |
|---------------|-----------------|--------------------------|
| 일지 | journal | 일지 |
| 점검 일정 | inspection_schedule | 점검 |
| AS | as_request | AS |
| 고객 | customer | 고객 |
| 장비 | equipment | 장비 |
| 담당자 | assignee | 담당자 |

- **실무 유지:**  
  - 입력 필드는 위 1~2절처럼 통일하고,  
  - 저장은 DB(또는 JSON)로 하되,  
  - 엑셀은 “시트별 내보내기”로 기존 업무 방식과 호환하면 된다.  
- **확장:**  
  - 검색·필터·월별은 모두 위 엔티티와 필드를 기준으로 쿼리/뷰를 설계하면 통계·월별 보고서로 자연스럽게 확장 가능하다.

이 문서를 “데이터/엑셀 탭”의 입력·저장·조회 설계 기준으로 사용하면 된다.
