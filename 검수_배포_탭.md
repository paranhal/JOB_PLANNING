## 3. 실행 파일 패키징 방식 제안

### 3-1. 권장: PyInstaller

| 항목 | 내용 |
|------|------|
| **용도** | Windows `.exe`, macOS 실행 파일(또는 `.app`) 생성 |
| **설치** | `pip install pyinstaller` |
| **단일 파일 예시** | `pyinstaller --onefile --windowed main.py` |
| **폴더 형태 예시** | `pyinstaller --onedir --windowed main.py` (디버깅·수정 시 유리) |
| **아이콘 지정** | `--icon=경로/아이콘.ico` (Windows), `.icns` (macOS) |
| **주의** | 데이터 파일·리소스는 `--add-data`로 포함하거나, 실행 경로 기준 상대 경로로 로드해야 함 |

### 3-2. 대안

| 도구 | 특징 | 사용 시기 |
|------|------|-----------|
| **PyInstaller** | 가장 많이 씀, 문서 많음 | 기본 선택 |
| **cx_Freeze** | 라이선스 제약 적음 | PyInstaller 이슈 시 |
| **Nuitka** | 컴파일 방식, 실행 속도·보안 유리 | 고급 최적화 필요 시 |

### 3-3. 패키징 체크리스트

| # | 확인 항목 | 체크 |
|---|-----------|:----:|
| P-1 | `main.py`(또는 진입점 스크립트) 경로가 PyInstaller 명령에 맞게 설정되어 있는가? | [ ] |
| P-2 | GUI 앱인 경우 `--windowed`로 콘솔 창이 뜨지 않게 했는가? | [ ] |
| P-3 | DB/설정 파일 등 리소스는 `--add-data` 또는 실행 위치 기준 경로로 처리했는가? | [ ] |
| P-4 | 빌드 후 다른 PC(또는 가상 환경)에서 실행해 보았는가? | [ ] |

## 4. 배포 시 주의할 점

### 4-1. 파일·경로

| # | 항목 | 주의 내용 | 체크 |
|---|------|-----------|:----:|
| D-1 | **실행 파일 위치** | 패키징 시 `sys.executable` 또는 `os.getcwd()` 차이 있음. 데이터/설정은 “실행 파일이 있는 폴더” 또는 “사용자 홈/문서” 기준으로 두는 것을 권장 | [ ] |
| D-2 | **상대 경로** | 개발 시 `./data` 등은 “현재 작업 디렉터리” 기준. exe 실행 시 CWD가 다를 수 있으므로 `os.path.dirname(sys.executable)` 등으로 기준 경로 고정 | [ ] |
| D-3 | **DB/설정 파일 위치** | 예: `실행파일과 같은 폴더/data/app.db` 또는 `~/Documents/업무일지/data/app.db` — 한 가지 방식으로 통일하고 문서화 | [ ] |

### 4-2. 아이콘·리소스

| # | 항목 | 주의 내용 | 체크 |
|---|------|-----------|:----:|
| D-4 | **아이콘** | Windows: `.ico`, macOS: `.icns` 준비 후 `--icon`으로 지정 | [ ] |
| D-5 | **리소스 포함** | 이미지·설정 템플릿 등은 `--add-data "원본경로;대상경로"`(Windows) 또는 `:`(macOS)로 포함 | [ ] |

### 4-3. 설정·데이터

## 5. Git commit 메시지 후보 & 작업일지 요약

### 5-1. Git commit 메시지 후보

배포 단계에 맞춰 하나 골라 사용하면 된다.

- `docs: 검수·배포 탭 점검 체크리스트 및 배포 가이드 추가`
- `docs: MVP 점검 체크리스트, 테스트 항목, PyInstaller 배포 가이드 정리`
- `chore: 배포 전 점검용 검수·배포 문서 추가 (체크리스트, 패키징, 주의사항)`

### 5-2. 작업일지용 요약 문장 (한 줄)

- **문장 1:** “업무일지 프로그램 검수·배포 탭 문서를 작성하여, 배포 전 점검 체크리스트, 입력/저장/수정/조회 테스트 항목, PyInstaller 패키징 방식, 배포 시 경로·아이콘·설정·데이터 저장 위치 등 주의사항을 정리하고, Git commit 메시지 후보와 작업일지 요약 문장을 추가했다.”
- **문장 2 (짧은 버전):** “검수·배포 탭에 점검 체크리스트, 기능 테스트 항목, 실행 파일 패키징 가이드, 배포 시 주의사항을 정리하고 commit/작업일지용 문구를 작성했다.”
