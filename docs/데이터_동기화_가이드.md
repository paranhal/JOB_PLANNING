# Windows · 맥 미니 · 맥북에어 — 데이터 수동 동기화 가이드

윈도우, 맥 미니, 맥북에어에서 개발·구현 시 **실데이터(또는 개발용 데이터)를 동일하게 맞추고 싶을 때**, 직렬화(export) / 역직렬화(import)로 **수동 동기화**하는 방법을 정리한 문서입니다.

---

## 1. 개념 정리

| 용어 | 의미 | 이 프로젝트에서 |
|------|------|------------------|
| **직렬화 (Serialization)** | 메모리/DB 데이터를 **한 파일(또는 묶음)**로 내보내기 | Export: DB/JSON → 하나의 이식 가능한 파일(또는 zip) |
| **역직렬화 (Deserialization)** | 그 파일을 읽어서 **다시 DB/로컬 데이터로 복원** | Import: 파일 → DB/JSON 복원 |
| **수동 동기화** | 한 기기에서 Export → 파일 복사 → 다른 기기에서 Import | 자동 서버 동기화 없이, 필요할 때 사람이 옮기는 방식 |

목표: **한 기기에서 만든 실데이터(또는 개발 데이터)를 다른 기기에서 그대로 쓸 수 있게 하기.**

---

## 2. 현재 데이터가 있는 곳

| 환경 | 저장 방식 | 위치 |
|------|------------|------|
| **as_support** (Python GUI) | JSON 파일 (엔티티별) | `data/*.json` (code_master.json, customer_master.json 등) |
| **server** (Go 웹) | SQLite | `server/data/app.db` |

두 저장소가 **다르기 때문에**, 동기화 전략을 둘 중 하나로 정해야 합니다.

- **A안:** “데이터 원천을 하나로” — 예: 서버(SQLite)를 기준으로 하고, as_support는 서버 API로 읽기/쓰기 (구현 범위 큼).
- **B안:** “내보내기/가져오기 파일로 동기화” — 각 환경에서 **Export → 파일 복사 → Import** 로 수동 동기화 (구현 단순, 지금 제안).

아래는 **B안(수동 동기화)** 기준입니다.

---

## 3. 수동 동기화 절차 (공통 흐름)

```
[기기 A: 예) 맥북에어]
  1) 개발/실데이터 입력
  2) "데이터 내보내기(Export)" 실행 → snapshot 파일 1개 생성
  3) 그 파일을 기기 B, C로 복사 (USB, 클라우드, Git 등)

[기기 B / C: 예) 윈도우, 맥 미니]
  4) snapshot 파일을 프로젝트 안으로 복사
  5) "데이터 가져오기(Import)" 실행 → 기기 A와 동일한 데이터로 복원
```

- **Export = 직렬화**: 현재 DB/JSON을 한 번에 하나의 이식용 파일로 만든다.
- **Import = 역직렬화**: 그 파일을 읽어서 현재 환경의 DB/JSON을 덮어쓰거나 병합한다.

---

## 4. 환경별 Export / Import 방법

### 4-1. as_support (Python, JSON)

**저장 위치:** `data/*.json` (프로젝트 루트 또는 as_support 기준 data)

**지금 가능한 수동 동기화 (도구 없이)**

| 단계 | 할 일 |
|------|--------|
| Export | `data/` 폴더 전체를 zip으로 압축 (예: `data_snapshot_20250320.zip`) |
| 복사 | zip을 USB/클라우드/Git 등으로 다른 기기에 전달 |
| Import | 다른 기기에서 zip 압축 해제 후, 내용물을 해당 기기 프로젝트의 `data/` 폴더에 **덮어쓰기** |

**나중에 구현하면 좋은 기능 (앱 안에 버튼으로)**

- **내보내기:** `data/` 아래 모든 JSON을 읽어서 하나의 **JSON 묶음 파일** (예: `export.json` 또는 `export_YYYYMMDD.json`)로 저장.
- **가져오기:** 그 JSON 묶음을 읽어서 `data/` 아래 엔티티별 JSON으로 다시 씀 (전체 덮어쓰기 또는 병합 옵션).

직렬화 형식 예시 (단일 파일에 다 넣는 경우):

```json
{
  "version": "1.0",
  "exported_at": "2025-03-20T12:00:00",
  "code_master": [ ... ],
  "customer_master": [ ... ],
  "contact": [ ... ],
  "installation": [ ... ]
}
```

---

### 4-2. server (Go, SQLite)

**저장 위치:** `server/data/app.db`

**지금 가능한 수동 동기화**

| 단계 | 할 일 |
|------|--------|
| Export | `server/data/app.db` 파일을 그대로 복사 (예: `app_db_20250320.db`) |
| 복사 | 해당 파일을 다른 기기로 전달 |
| Import | 다른 기기에서 서버 중지 후, 받은 파일을 `server/data/app.db` 로 덮어쓰기 후 서버 재시작 |

**나중에 구현하면 좋은 기능**

- **Export API 또는 CLI:** SQLite 내용을 **JSON** 또는 **SQL 덤프**로 내보내기 (직렬화).
- **Import API 또는 CLI:** 그 JSON/SQL을 읽어서 SQLite를 복원(역직렬화).  
  → 그러면 “파일 하나”만 옮겨도 되고, 버전/스키마 호환도 관리하기 쉬움.

---

## 5. “한 번에” 동기화하고 싶을 때 (권장 포맷)

두 환경(as_support JSON + server SQLite)을 **한 번에** 맞추고 싶다면, **하나의 동기화 번들**을 쓰는 방법을 권장합니다.

- **이름 예:** `sync_bundle_YYYYMMDD.zip`
- **안에 넣을 것:**
  - `data/` — as_support용 JSON들 (또는 위의 단일 `export.json`)
  - `server_data/app.db` — 서버용 SQLite (또는 서버 Export JSON/SQL)

**수동 절차 요약**

1. **기기 A (예: 맥북에어)**  
   - as_support: `data/` 폴더 복사  
   - server: `server/data/app.db` 복사  
   - 두 개를 한 폴더에 넣고 `sync_bundle_20250320.zip` 으로 압축
2. **파일 전달**  
   - USB, Google Drive, OneDrive, 또는 (민감하지 않다면) Git 저장소의 `sync/` 같은 폴더에 올리기
3. **기기 B, C (윈도우, 맥 미니)**  
   - zip 압축 해제  
   - `data/` → 프로젝트 `data/` 에 덮어쓰기  
   - `server_data/app.db` → `server/data/app.db` 로 덮어쓰기 (서버는 중지한 뒤에)

이렇게 하면 **직렬화(번들 생성) → 복사 → 역직렬화(압축 해제·덮어쓰기)** 로 세 기기 데이터를 수동 동기화할 수 있습니다.

---

## 6. 주의사항

- **덮어쓰기:** Import는 보통 **덮어쓰기**이므로, 받는 쪽에만 있는 데이터는 사라질 수 있음. 필요하면 Import 전에 `data/`, `app.db` 백업.
- **동시 편집:** 수동 동기화는 “한쪽에서 Export한 시점”을 기준으로 하므로, 두 기기에서 동시에 수정하면 나중에 Export한 쪽이 이김. 충돌 해소는 자동이 아니므로, “기준 기기 하나 정해서 그쪽에서만 Export” 하는 것을 권장.
- **민감 데이터:** 실데이터에 개인정보 등이 있으면 zip/파일을 Git에 올리지 말고, USB·암호 걸린 클라우드 등으로만 전달.

---

## 7. 요약

| 목적 | 방법 |
|------|------|
| **직렬화** | Export: 현재 데이터를 **이식용 파일 1개(또는 zip)** 로 만든다. |
| **역직렬화** | Import: 그 파일을 **다시 DB/JSON으로 복원**한다. |
| **수동 동기화** | 기기 A에서 Export → 파일 복사 → 기기 B/C에서 Import. |
| **as_support** | 지금은 `data/` 폴더 zip으로 Export/Import 가능; 추후 앱에 “내보내기/가져오기” 버튼 추가 권장. |
| **server** | 지금은 `server/data/app.db` 복사로 Export/Import 가능; 추후 JSON/SQL Export·Import 기능 추가 권장. |
| **한 번에 맞추기** | `sync_bundle_YYYYMMDD.zip` 에 `data/` + `app.db` 넣어서 세 기기 간에 같은 절차로 복사·덮어쓰기. |

이 가이드를 기준으로, 나중에 각 앱에 “데이터 내보내기 / 데이터 가져오기” 기능을 넣으면, 직렬화·역직렬화를 통해 수동 동기화를 더 편하게 할 수 있습니다.
