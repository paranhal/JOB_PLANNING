# JOB_PLANNING 에이전트 5종 사용법

**현재:** 기획서 기준으로 **재구축** 단계. 모든 설계·구현의 **유일 기준**은 `기획_업무일지_프로그램_요구사항.md`(기획서)입니다.

이 프로젝트는 역할별 에이전트 5개를 Cursor 규칙으로 운영합니다. 채팅에서 **해당 규칙을 @로 붙이면** 그 역할로만 동작합니다.

---

## 기준 문서 (단일)

| 문서 | 역할 |
|------|------|
| **`기획_업무일지_프로그램_요구사항.md`** | 기획·요구사항의 **유일 기준**. 구현/DB/테스트/패치 모두 이 기획서를 따름. |

---

## 호출 방법

### 1) 채팅에서 @로 규칙 붙이기 (권장)

1. Cursor 채팅(Cmd+L)을 연다.
2. 입력창에 **@** 를 친다.
3. **Rules**에서 원하는 에이전트 규칙을 선택한다. (예: `agent-planning`, `agent-implementation`)
4. 질문/지시를 입력한다.

### 2) 관련 파일을 열어두기

- 기획서·기획 문서 → 기획 에이전트
- 구현 설계·코드 → 구현 에이전트
- 데이터·DB 문서 → DB 설계 에이전트
- 테스트 파일 → 테스트 에이전트
- 검수·배포 문서 → 패치 에이전트

### 3) 채팅을 역할별로 나누기

채팅마다 한 가지 에이전트만 @로 붙이면 역할이 섞이지 않습니다.
