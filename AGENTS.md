# JOB_PLANNING 에이전트 5종 사용법

이 프로젝트는 역할별 에이전트 5개를 Cursor 규칙으로 운영합니다. 채팅에서 **해당 규칙을 @로 붙이면** 그 역할로만 동작합니다.

---

## 에이전트 목록

| 역할 | 규칙 파일 | 담당 범위 |
|------|-----------|-----------|
| **기획** | `@.cursor/rules/agent-planning.mdc` | 요구사항, 기획서, 엑셀 분석 |
| **구현** | `@.cursor/rules/agent-implementation.mdc` | 코드·파일구조·탭 구현 |
| **DB 설계** | `@.cursor/rules/agent-db.mdc` | 스키마, 데이터·엑셀 구조 설계 |
| **테스트** | `@.cursor/rules/agent-test.mdc` | 단위/통합 테스트 작성 |
| **패치** | `@.cursor/rules/agent-patch.mdc` | 버그수정, 핫픽스, 배포·검수 |

---

## 호출 방법

### 1) 채팅에서 @로 규칙 붙이기 (권장)

1. Cursor 채팅(Cmd+L)을 연다.
2. 입력창에 **@** 를 친다.
3. **Rules** 또는 **.cursor/rules** 에서 원하는 에이전트 규칙 파일을 선택한다.
   - 예: `agent-planning` → 기획 에이전트
   - 예: `agent-implementation` → 구현 에이전트
4. 그 다음 질문/지시를 입력한다.

이렇게 하면 해당 채팅에서는 선택한 에이전트 역할만 유지합니다.

### 2) 관련 파일을 열어두기

특정 파일을 열어두면 **그 파일에 맞는 규칙이 자동으로 적용**될 수 있습니다.

- 기획 문서(`기획_업무일지_프로그램_요구사항.md`, `엑셀_시트_분석결과.md` 등) → 기획 에이전트
- 구현 설계·코드(`구현_탭_파일구조_설계.md`, `*.py` 등) → 구현 에이전트
- 데이터·DB 문서(`데이터_*.md`) → DB 설계 에이전트
- 테스트 파일(`test_*.py`, `tests/`) → 테스트 에이전트
- 검수·배포 문서(`검수_배포_탭.md` 등) → 패치 에이전트

### 3) 채팅을 역할별로 나누기

- 기획만 할 채팅 → `@agent-planning.mdc` 만 붙이고 기획 작업
- 구현만 할 채팅 → `@agent-implementation.mdc` 만 붙이고 구현 작업  
이렇게 **채팅마다 한 가지 에이전트만** 쓰면 역할이 섞이지 않습니다.

---

## 요약

- **기획** → `@agent-planning.mdc` 또는 기획 관련 md 열기  
- **구현** → `@agent-implementation.mdc` 또는 구현/코드 파일 열기  
- **DB 설계** → `@agent-db.mdc` 또는 데이터 관련 md 열기  
- **테스트** → `@agent-test.mdc` 또는 테스트 파일 열기  
- **패치** → `@agent-patch.mdc` 또는 검수·배포 문서 열기  

규칙 파일 위치: `.cursor/rules/agent-*.mdc`
