# 업무일지 프로그램 기획서

**목적:** 엑셀 기반 업무일지를 대체·보완하는 파이썬 기반 실행형 도구 (개인/소규모 실무용)

**업무 맥락:** 도서관 자동화 장비·LAS 영업 및 현장 기술 지원 — 정기 점검, 담당자, AS 요청·대응 관리 → 고객/장비 이력·고객별 보유 장비까지 확장

**분석 대상:** `2025년_비젼아이티_유지보수현황(06161800).xlsx` (실제 업무용 엑셀 기준 반영)

### 0-3. 프로그램에서 통합·구분 방안

- **처리 이력:** NICOM 처리리스트 + 원콜통합 + 세종 KLAS 처리리스트 → 하나의 “처리 이력” 모델로 통합, `구분`(NICOM/원콜/KLAS 등) 또는 `출처`로 필터.
- **고객·담당자·장비:** NICOM_유지보수현황, VISIONIT_MaintenanceList, 충남교육청_KLAS, 세종시도서관_KLAS → “고객 마스터” + “담당자” + “장비(고객별)” 로 통합 저장, 화면에서 구분(라인) 필터 가능하도록.
- **일정:** 1~12월 시트 → “월별 일정” 하나의 기능으로 제공(날짜별 방문처·메모).

## 3. MVP(1차 범위) 제안 — 현행 엑셀 시트 기준

**원칙:** “현재 쓰는 엑셀 시트를 대체할 수 있는 최소 기능” — 처리 이력(NICOM·원콜/KLAS) + 고객·담당자·장비 마스터 + 월별 일정 + 내보내기

### 3-1. MVP 포함 기능 (엑셀 시트와 매핑)

| 순위 | 기능 | MVP | 대응 시트 | 비고 |
|------|------|-----|-----------|------|
| 1 | **처리 이력 (NICOM)** 일시·고객·담당자·모델·접수/처리내용·진행여부 | ✅ | NICOM_처리리스트 | 핵심 업무 기록 |
| 2 | **고객·담당자·장비 마스터** 고객명·구분·담당자·연락처·장비·점검주기·유무상 등 | ✅ | NICOM_유지보수현황, VISIONIT_MaintenanceList | 처리 이력에서 고객/장비 선택용 |
| 3 | **처리 이력 (원콜/KLAS)** 일자·처리일자·분류·제목·내용·진행·답변·도서관 | ✅ | 25년원콜지원센터통합, 세종시_KLAS 처리리스트 | 원콜·KLAS 대응 |
| 4 | **월별 일정** 날짜별 방문처·고객 목록(텍스트) | ✅ | 1월~12월 | 주간 일정 뷰 |
| 5 | **기간별 처리 이력 조회·수정·삭제** | ✅ | — | 공통 |
| 6 | **엑셀 내보내기** (시트별 또는 통합) | ✅ | — | 백업·기존 양식 호환 |
| 7 | **고객/장비 이력 통합 조회** | ⬜ 2차 | — | MVP 이후 |
| 8 | **NICOM_유지보수_통계, 2025예산** | ⬜ 2차 | 통계·예산 시트 | 확장 |

### 3-2. MVP에서 의도적으로 제외·축소

- **NICOM_유지보수_통계** 시트 → 2차 (통계/대시보드)
- **AS** 시트(지역별 요약표) → 참고용; 필요 시 2차에 “AS 요약” 기능으로 반영
- **충남교육청_KLAS, 세종시도서관_KLAS** → MVP에서는 “고객 마스터”에 구분(라인)으로 통합 저장 후 필터로 조회
- 고객/장비 **상세 이력 통합 화면** → 2차
- **엑셀 가져오기** → 2차
- **다음 점검일 자동 계산** → 2차 선택 기능
- 다중 사용자·권한, 서버·클라우드 → 제외

## 요약

- **현행 엑셀:** 처리 이력(NICOM_처리리스트, 원콜통합, 세종 KLAS), 고객·담당자·장비(유지보수현황, VISIONIT, 충남/세종 KLAS), 월별 일정(1~12월), 통계·예산·AS 요약 시트 등 22개 시트 사용.
- **핵심:** 처리 이력(NICOM + 원콜/KLAS) + 고객·담당자·장비 마스터 + 월별 일정 + 로컬 저장 + 엑셀 내보내기.
- **MVP:** 처리 이력 2종 입력·조회·수정 + 고객·담당자·장비 마스터 + 월별 일정 + 엑셀 내보내기. 통계·예산·AS 요약·이력 통합은 2차.
- **저장:** 로컬 DB(SQLite) 또는 JSON 세트, 엑셀은 “내보내기”로 기존 양식 호환.
- **확장:** NICOM 통계, 예산, 고객/장비 이력 통합, 엑셀 가져오기, AS 요약, 알림, 실행 파일 패키징 순.

이 기획서와 `엑셀_시트_분석결과.md`를 기준으로 1단계(데이터 구조 + 저장)부터 구현하면, 점진적으로 현재 엑셀 업무일지를 대체·보완할 수 있다.
